# Anchor Compiler Schema Plan

## 1. Goal
Design the deterministic schema mapping the 7 dimensions of `dip_state.yml` into the physical `GEMINI.md` anchor.

## 2. Mapping Strategy
`GEMINI.md` will consist of three sections:
1. **The Projection Header:** A clear `> DO NOT EDIT. This is a materialized projection from dip_state.yml` warning.
2. **The Universal Shim:** The existing `GEMINI.md` platform shim that mandates reading `commons/AGENT.md`.
3. **The Dyad Instance State:** A compiled section where each dimension from `dip_state.yml` is rendered as a Markdown section. Only dimensions with `status: APPROVED` will be compiled into the anchor.

## 3. Template Example
```markdown
# dyad-cairn — GEMINI.md
```
