# Reflect: Excising Dead Mass (The Frontier Pruning)

## Continue
We must continue aggressively shrinking our operational surface area. The Operator's mandate to question the value of `DONE` frontier nodes highlights our commitment to radical parsimony.

## Start
We must start physically excising completed (`DONE`) nodes from the active DAG (`artifacts/frontier_state.yml`). Once a node is materialized and merged, it is historically sealed in the `DYAD_LEDGER.md` and the Touchstone Retros. Leaving it in the active tracking layer is a cognitive and structural bloat.

## Stop
We must stop allowing historical artifacts to accumulate in active execution layers. A frontier only has edges. Once space is conquered, it is no longer the frontier. We must stop projecting completed state into the active view.
